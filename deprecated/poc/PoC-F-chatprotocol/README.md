# PoC-F: `smith://` チャットトランスポートの検証

**目的**: Mod用RPCが存在しないPalworldで、チャットメッセージを
クライアント↔サーバー通信路として使えるか確認する。成立すれば
`server_action`(⑤)とハンドシェイク(F5)の土台になる。

## 検証項目

1. **往復**: クライアントがチャットで `smith://ping/<任意文字列>` を送信 →
   サーバー側フックが受信 → `SendSystemAnnounce` で送信者に応答が返るか
2. **表示抑制**: `smith://` メッセージを他プレイヤーのチャット欄に**表示させない**ことができるか
   - 実験1: フック内で `msg.Message` を書き換えたら表示も変わるか(=改変可能か)
   - 実験2: 空文字に書き換えたら実質非表示にできるか
   - ここが不成立でも機能はする(プロトコルが見えて不格好なだけ)
3. **ペイロード制限**: チャットの最大長・使える文字(JSONをそのまま流せるか、エスケープが要るか)

## 手順

1. `src/server/` を サーバーの `Pal/Binaries/Win64/ue4ss/Mods/PalSmithProtocol/` に配置し有効化
2. クライアントから通常チャットで `smith://ping/hello` を送信
3. システムアナウンスで `smith://pong/hello` が返ることを確認
4. 別クライアントのチャット欄に元メッセージが見えるか観察(抑制実験の結果を記録)
